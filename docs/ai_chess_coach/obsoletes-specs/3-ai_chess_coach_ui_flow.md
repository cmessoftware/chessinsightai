# ChessTrainer AI Coach – UI Integration Flow

## Objective

Define the UI and backend interaction flow for generating an AI coaching report when a user uploads a PGN.

The goal is to simplify the user experience:

User uploads a PGN → system automatically analyzes the game → generates coaching feedback.

No manual steps required.

---

# Current Situation

Current UI features:

- PGN upload page
- button for feature generation
- dashboard with metrics
- work in progress for natural language analysis

Proposed improvement:

The system should automatically execute the AI coaching pipeline immediately after PGN upload.

---

# Target User Flow

User opens the PGN analysis page.

Steps:

1 User uploads PGN file

2 Backend automatically performs:

- feature extraction
- ML evaluation
- RAG retrieval
- LLM report generation

3 UI displays results:

- coaching report
- metrics dashboard
- detected patterns

# Backend Pipeline

When a PGN is uploaded the backend must trigger the following pipeline.

PGN upload  
→ parse game  
→ run Stockfish analysis  
→ generate ML features  
→ summarize features  
→ retrieve chess knowledge (RAG)  
→ generate coaching report  
→ store results

---

# Backend Service Flow

Endpoint example:

POST /analyze_game

Input:

PGN file

Process:

1 parse PGN
2 run Stockfish analysis
3 compute features
4 summarize analysis
5 retrieve chess knowledge
6 generate report with LLM
7 store report and metrics

Output:

analysis_id

# UI Components

The UI should have three main sections.

Natural language explanation generated by the AI.

Sections:

Strengths  
Weaknesses  
Improvement suggestions  

Example:

Strengths

Weaknesses
- King safety weakened after move 20.

Improvement suggestions
- Study defensive responses to knight forks.
