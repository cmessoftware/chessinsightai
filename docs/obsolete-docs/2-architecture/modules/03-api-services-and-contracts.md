# ChessTrainer AI Coach – Structured Output Specification

## Objective

Define a structured output format for the AI coaching report generated by the LLM.

Using structured outputs reduces:

- hallucinations
- inconsistent formatting
- parsing errors

The LLM should always return a JSON object following a defined schema.

The UI will then render the information.

---

# Why Structured Output

Natural language output from LLMs can be inconsistent.

Example problem:

Model outputs different formats each time.

Example 1

Strengths:
- good center control

Example 2

Positive aspects of the game:
...

This makes parsing difficult.

Instead, enforce a JSON schema.

---

# Output Schema

The LLM must produce the following JSON structure.

```json
{
  "opening": "",
  "game_summary": "",
  "strengths": [],
  "weaknesses": [],
  "improvement_suggestions": [],
  "critical_moments": [],
  "training_topics": []
}

Field Definitions
opening

Opening played in the game.

Example

"Italian Game"
game_summary

Short overall description of the game.

Example

"White obtained central control in the opening but lost the initiative after a tactical mistake in the middlegame."
strengths

Positive aspects detected in the analysis.

Example

[
 "Good central control in the opening",
 "Active rook placement on open files"
]
weaknesses

Recurring problems detected.

Example

[
 "Weak king safety after castling",
 "Delayed development of queenside pieces"
]
improvement_suggestions

Concrete recommendations.

Example

[
 "Study defensive patterns against knight forks",
 "Improve coordination before launching attacks"
]
critical_moments

List of key turning points.

Example

[
 "Move 17: missed tactic allowing material loss",
 "Move 24: blunder leading to losing position"
]
training_topics

Suggested training themes.

Example

[
 "fork tactics",
 "king safety",
 "defensive calculation"
]
Prompt Strategy

The prompt must explicitly request JSON output.

Example instruction:

You are a chess coach.

Analyze the provided game summary and chess knowledge.

Return ONLY valid JSON following the schema below.

Do not include explanations outside JSON.

If information is missing use empty fields.

Prompt Template

System role:

You are an experienced chess coach.

You must generate structured coaching reports.

User prompt:

Game summary:
{feature_summary}

Relevant chess knowledge:
{retrieved_documents}

Return JSON using the schema:

{
"opening": "",
"game_summary": "",
"strengths": [],
"weaknesses": [],
"improvement_suggestions": [],
"critical_moments": [],
"training_topics": []
}

Do not include text outside the JSON.

Validation Layer

After the LLM generates the response, the backend should validate the structure.

Validation steps:

1 check valid JSON
2 check required fields
3 enforce list types
4 fallback if invalid

Example fallback strategy:

retry generation

simplify prompt

return partial report

UI Rendering

The frontend should map JSON fields to UI sections.

Mapping example:

opening → Game summary card
game_summary → introduction text
strengths → strengths panel
weaknesses → weaknesses panel
improvement_suggestions → recommendations section
critical_moments → key positions viewer
training_topics → training suggestions

Example Final JSON
{
  "opening": "Italian Game",
  "game_summary": "White had good central control early but allowed a tactical mistake in the middlegame.",
  "strengths": [
    "Good central pawn control",
    "Active bishop development"
  ],
  "weaknesses": [
    "King safety weakened after move 20",
    "Missed defensive tactic"
  ],
  "improvement_suggestions": [
    "Study defensive resources against forks",
    "Improve calculation before committing attacks"
  ],
  "critical_moments": [
    "Move 17: missed fork tactic",
    "Move 24: blunder losing material"
  ],
  "training_topics": [
    "fork tactics",
    "king safety"
  ]
}
Integration in Pipeline

Pipeline with structured output.

PGN
→ Stockfish analysis
→ feature extraction
→ feature summarizer
→ RAG retrieval
→ prompt builder
→ LLM (JSON output)
→ validation
→ UI rendering

Advantages

Structured output enables:

deterministic UI rendering

easier debugging

easier dataset creation

improved reliability

It also allows storing coaching reports in the database for later analysis.

Future Extensions

Possible improvements:

- score confidence values
- attach board positions to critical moments
- generate training exercises automatically
- compare reports across multiple games

from pydantic import BaseModel, Field, validator
from typing import Optional, Literal

class AnalysisOptions(BaseModel):
    """Analysis configuration options."""
    
    depth: int = Field(
        default=20,
        ge=10,
        le=40,
        description="Stockfish analysis depth"
    )
    
    enable_ml: bool = Field(
        default=True,
        description="Enable ML prediction"
    )
    
    enable_rag: bool = Field(
        default=True,
        description="Enable RAG retrieval"
    )
    
    enable_cv: bool = Field(
        default=False,
        description="Enable CV extraction"
    )
    
    elo_threshold: Optional[int] = Field(
        default=None,
        ge=800,
        le=3000,
        description="Player ELO for adaptation"
    )
    
    focus_mode: Literal["critical", "full", "tactical", "positional"] = Field(
        default="critical",
        description="Modo de enfoque"
    )
    
    @validator("depth")
    def depth_must_be_multiple_of_5(cls, v):
        """Depth must be a multiple of 5 for efficiency."""
        if v % 5 != 0:
            raise ValueError("depth must be multiple of 5")
        return v
    
    class Config:
        schema_extra = {
            "example": {
                "depth": 20,
                "enable_ml": True,
                "enable_rag": True,
                "enable_cv": False,
                "elo_threshold": 1800,
                "focus_mode": "critical"
            }
        }
```

from pydantic import BaseModel, Field, validator
from typing import List, Dict, Optional, Any, Literal
from datetime import datetime
from uuid import UUID

class MLPrediction(BaseModel):
    """ML model prediction."""
    
    predicted_error: Literal["good", "inaccuracy", "mistake", "blunder"]
    confidence: float = Field(ge=0.0, le=1.0)
    risk_score: float = Field(ge=0.0, le=1.0)
    contributing_features: List[Dict[str, Any]] = Field(default_factory=list)

class RAGContext(BaseModel):
    """Context retrieved from RAG."""
    
    similar_positions: List[Dict] = Field(default_factory=list)
    book_excerpts: List[str] = Field(default_factory=list)
    player_patterns: List[Dict] = Field(default_factory=list)
    total_retrieved: int = Field(ge=0)
    relevance_scores: List[float] = Field(default_factory=list)

class ExecutionResult(BaseModel):
    """Execution result for a single move analysis."""
    
    # Identification
    game_id: UUID
    ply: int = Field(ge=0)
    move_san: str
    fen_before: str
    fen_after: str
    
    # Engine Evaluation
    engine_eval_before: float
    engine_eval_after: float
    score_diff: float
    best_move: str
    best_line: List[str] = Field(default_factory=list)
    
    # Features
    features: Dict[str, float] = Field(default_factory=dict)
    tactical_tags: List[str] = Field(default_factory=list)
    phase: Literal["opening", "middlegame", "endgame"]

    @validator("score_diff", always=True)

        return v

from typing import List, Dict, Any, Literal

class ValidationIssue(BaseModel):

    issues: List[ValidationIssue] = Field(default_factory=list)
    passed_rules: List[str] = Field(default_factory=list)
    failed_rules: List[str] = Field(default_factory=list)
    
    @validator("is_consistent", always=True)
    def consistent_if_no_errors(cls, v, values):
        """is_consistent debe ser False si hay issues con severity=error."""
        issues = values.get("issues", [])
        has_errors = any(issue.severity == "error" for issue in issues)
        
        if has_errors and v:
            raise ValueError("Cannot be consistent with error-level issues")

from pydantic import BaseModel, Field
from typing import List, Dict, Any
from uuid import UUID

class AnalysisReport(BaseModel):
    """Complete game analysis report."""
    
    game_id: UUID
    player_id: int
    total_moves_analyzed: int = Field(ge=0)
    enriched_results: List[EnrichedResult]
    player_patterns: Optional["PlayerPatterns"] = None
    metadata: Dict[str, Any] = Field(default_factory=dict)
    
    @property
    def consistency_rate(self) -> float:
        """% de resultados consistentes."""
        if not self.enriched_results:
            return 0.0
        consistent = sum(1 for r in self.enriched_results if r.critic_result.is_consistent)
        return consistent / len(self.enriched_results)
    
    @property
    def avg_confidence(self) -> float:
        """Confianza promedio."""
        if not self.enriched_results:
            return 0.0
        confidences = [r.critic_result.confidence for r in self.enriched_results]
        return sum(confidences) / len(confidences)
    
    class Config:
        schema_extra = {
