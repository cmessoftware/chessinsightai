# ChessTrainer AI Coach – UI Integration Flow

## Objective

Define the UI and backend interaction flow for generating an AI coaching report when a user uploads a PGN.

The goal is to simplify the user experience:

User uploads a PGN → system automatically analyzes the game → generates coaching feedback.

No manual steps required.

---

# Current Situation

Current UI features:

- PGN upload page
- button for feature generation
- dashboard with metrics
- work in progress for natural language analysis

Proposed improvement:

The system should automatically execute the AI coaching pipeline immediately after PGN upload.

ChessTrainer — Kasparov Decision Engine — Functional Design

## Objective

Implement a game analysis module that captures the **player's decision-making process**, not just the move outcome.

The module must:

- detect **critical moments**
- reconstruct **plans and candidates**
- compare against the **engine**
- classify **error types**
- generate **ML features**

This module will feed:

- dashboards
- automatic exercise generation
- training datasets
- personalized recommendations

---

Each game will follow this flow.


PGN

# Step 1 — Critical Moment Detection

A critical moment is a move where:

- large evaluation change
- relevant strategic decision
- attack initiation
- significant simplification
- serious error

## Heuristics

Detect if:


abs(score_diff) > threshold


o


abs(depth_score_diff) > threshold


Also mark moves where:

- a tactic appears
- phase change
- major piece capture

## Output


critical_positions


Ejemplo


move_number
fen
score_before
score_after
phase


---

# Step 2 — Plan Reconstruction

Attempt to infer the player's plan from the position.

## Features Used

- pawn structure
- open files
- king safety
- material
- piece activity

## Plan Types


attack_king
central_break
minority_attack
positional_pressure
endgame_transition
piece_improvement
defensive_hold


## Output


plan_type
plan_confidence


---

# Step 3 — Candidate Move Generation

For each critical position generate **at most 3 candidates**.

## Method

1. player's move
2. engine's best move
3. best strategic alternative

## Structure


candidate_moves


Ejemplo


[
{"move":"Qg4", "type":"played"},
{"move":"Qd2", "type":"engine_best"},
{"move":"Rfd1", "type":"strategic"}
]


---

# Step 4 — Engine Evaluation

Evaluate each candidate.

Engine:


Stockfish


Stored data


score_cp
mate_in
depth
principal_variation


---

# Step 5 — Error Classification

Compare the player's move against the engine's best.

## Categories


brilliant
good

mistake
blunder


## Additional Classification
